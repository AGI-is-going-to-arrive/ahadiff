三、护城河

五个没人占的山头

把 12 款产品的功能列齐，这 5 个格子全是空的 — AhaDiff 同时占住这 5 个格子，就是 100% 的差异化。

MOAT 01

Diff → 学习 闭环

不是背例题，而是拿你刚做错的题做三道相似题、两周后再考一遍。
你改了 src/auth/oauth.py
→ 生成 4 条带证据的笔记
→ 生成 5 道主动回忆 quiz
→ 14 天后 SRS 自动推送给你复习
v0.1 通过 ahadiff serve 实现浏览器内交互（Quiz/SRS 答题通过 REST API 写回 review.sqlite），不再需要在 terminal 和浏览器间来回切换。
CodeRabbit 只评审 Execute Program 只教预制课 AhaDiff 两头都做
MOAT 02

Claim → 证据 5 状态

论文审稿 — 每个结论都必须引原文行号，拒绝"感觉对"。
一条解释会被打上 5 种标签之一：
● verified — 能回到 diff 的某行
● weak — 有线索但不完整
● not_proven — 如"性能提升 20%"（diff 不含基准测试）
● contradicted — 跟 diff 相矛盾
● rejected — 引用了 patch 外的文件或不存在的证据
别家都是"自然语言带行号" AhaDiff 独家 5 状态机
MOAT 03

Git 棘轮（单调变好）

钢琴考级 — 新演奏必须比上次高分才能通过，否则退回原处。
Round 1: wiki_score = 0.72 ✓ git commit
Round 2: wiki_score = 0.84 ✓ git commit
Round 3: wiki_score = 0.76 ✗ 删除 worktree（不触碰主分支）
连续 2 个优化目标无增益 → 触发 Phase 2.5 探索性重写

8 维自研 RUBRIC (100 分)
accuracy 20 · evidence 18 · diff_coverage 14 · learnability 14
quiz_transfer 10 · spec_alignment 10 · conciseness 8 · safety_privacy 6
调研的 12 款产品中未发现类似机制 autoresearch 有但用于 ML AhaDiff 用于笔记质量
MOAT 04

Local-First 隐私

写日记锁在家里抽屉，而不是发到云端 SaaS。
per-repo 真相源存 .ahadiff/ 目录（review.sqlite / concepts.jsonl / audit.jsonl / runs/）
全局目录仅做派生索引和偏好，不参与 ratchet 判定
出本机的只有 LLM API prompt 本身（脱敏后）
脱敏顺序（不可跳过）：raw → scan → redact → 才能 log/cache/model/render
五道安全闸门：.ahadiffignore → secret scan(两层扫描) → UNTRUSTED_DIFF 边界（6 类） → 隐私三档 → audit.jsonl
Lesson HTML 经 bleach 过滤防 XSS
CodeRabbit / Greptile / DeepWiki 全 SaaS AhaDiff 纯本地
MOAT 05

i18n 学习笔记多语言

课本有中文版也有英文版，但笔记只有一种语言？AhaDiff 让笔记跟着你说的语言走。
优先级链：cookie ahadiff_lang → navigator.language → CLI --lang → config.toml → $LANG → 降级 en
LLM prompt 用 OUTPUT_LANGUAGE 指令前缀（单 prompt，不分语言文件）
概念用英文 term_key 稳定身份 + 本地化 display_name
SRS 卡片保留创建时语言，不重翻译
VCR cassette key 包含 output_lang，换语言不命中旧缓存
10 款竞品均无 diff→学习笔记的多语言生成能力（竞品矩阵已验证） AhaDiff 中英双语全链路（en + zh-CN）
三 · 附 工程护城河

竞品做不到的 5 个工程细节

这些不是"功能点"而是"安全/并发/部署"维度的深水区 — SaaS 竞品的架构决策让它们天然无法复刻。

ENG 01

三层锁并发安全

三把不同的钥匙管三扇门：repo 文件、数据库、serve 端口，互不干扰又不冲突。
repo_write — portalocker 跨平台文件锁（CLI learn/improve 用，PID 仅诊断）
db_write — SQLite WAL mode + busy_timeout=5000ms（CLI/serve 共享）
serve_write — Starlette asyncio.Lock 进程内排他

CLI 和 serve 可同时运行，写入安全由三层保护（从外到内获取），无需用户手动协调。
SaaS 竞品不需本地并发 AhaDiff local-first 刚需
ENG 02

Progressive Enhancement

有网看在线视频，没网照样看下载好的。两个模式用同一模板，体验一致。
file:// 静态模式 — 双击 HTML 即可阅读笔记（只读，Quiz 按钮提示 CLI 命令）
ahadiff serve — 本地 Starlette + Uvicorn，Quiz/SRS 答题写回 review.sqlite

同一 Jinja2 模板自动检测：有 serve → REST API 交互；无 serve → 优雅降级显示命令提示。
竞品全依赖云端 AhaDiff 断网可用
ENG 03

隐私三档分级

手机通讯录导出有三档：永不上传 / 脱敏后上传 / 明确授权后上传。
strict_local — 仅本地模型，数据不出机器（默认）
redacted_remote — 脱敏后发远端 LLM API
explicit_remote — 用户显式授权发原文

CLI 参数、config.toml、audit 日志统一 snake_case 命名。redaction_map.json 的 original_path 仅 strict_local 写入。
CodeRabbit/Greptile/DeepWiki 全 SaaS 无分档 AhaDiff 用户完全掌控
ENG 04

UNTRUSTED_DIFF 边界

不仅怀疑包裹里的东西，连包裹上的地址标签、快递员说的话都验。
不可信输入面覆盖 6 类：
diff 正文 · 文件名 · commit message
Graphify label · 模型输出 · VCR cassette

统一经 redaction_pipeline() 处理
顺序：raw → scan → redact → 才能 log/cache/model/render
竞品只过滤 diff 正文 AhaDiff 全链路防注入
ENG 05

Serve 安全模型

家里的 Wi-Fi 路由器 — 只在家门内可见，外面看不到也连不上。
bind=127.0.0.1 — 只监听本机回环（外部不可访问）
Host + Origin 双校验 — 防 DNS rebinding
读写鉴权分级：默认只读，写操作需 write token
Lesson HTML 经 bleach 过滤防 XSS
SaaS 竞品用云端鉴权 AhaDiff 本地零信任

THE GAP BETWEEN TWO WORLDS
附录 · Karpathy LLM Wiki 映射

LLM Wiki 思想的具体落地

Karpathy 2026 年 4 月提出的 persistent compounding wiki 模式，AhaDiff 如何将其适配到代码学习场景。
Karpathy 原版 核心描述 AhaDiff 落地 差异说明
输入源 文档、论文、网页 git diff（6 种输入模式） 源从静态文档变为代码变更，每次 diff 是一个增量知识单元
Ingest 操作 LLM 读源 → 提取 → 整合到 wiki ahadiff learn 增加了 claim 验证层（5 态枚举），确保每句解释有代码证据
index.md 内容导航页，LLM 每次 ingest 更新 .ahadiff/index.md（v0.2） diff-aware merge：只追加新概念或更新引用计数，不覆盖
log.md 时间线日志，append-only concepts.jsonl（v0.1） 结构化 JSONL，term_key-keyed upsert log + branch-aware 过滤（git ancestry 判断概念可见性）
Query 反写 好的回答反写到 wiki quiz/review → learning signals 学习行为（答题、标错）反馈到概念系统，驱动 improve loop
Lint（健康检查） 手动请求 LLM 检查矛盾/过时 evaluator.py + claim verifier 自动化 8 维评分 + 硬门禁，无需手动触发
知识复利 wiki 越来越丰富 concepts.jsonl 越学越厚 + Graphify repo map 底层 + 学习覆盖层上层 = 双层复利
Graphify × LLM Wiki 互补关系：Graphify 做 repo-level 全局 map（底层结构图谱，偶尔更新），AhaDiff 的 LLM Wiki 模式做 diff-level 增量 learning overlay（上层学习覆盖层，每次 learn 都增长）。两者通过 ahadiff graph import 连接，Graphify 节点成为 Concept/Claim 节点的锚点。Graphify 可选不强制，不安装时静默降级为"仅学习节点"模式。

六 · 附

v0.1 设计升级（2026-04-20 三轮评估后新增）

经三模型 + Explorer 五轮交叉审查，v0.1 取消静态 viewer 冻结，新增以下关键能力。
技术栈确认：Python 3.11+ / Typer / Rich / Pydantic / Jinja2 / Starlette+Uvicorn，前端用 Jinja2 模板 + Vanilla JS，SQLite 为唯一真相源。

serve 模式
ahadiff serve — 轻量本地服务器（Starlette + Uvicorn），Quiz/SRS 答题在浏览器内完成，结果直接写回 review.sqlite（唯一真相源），不用在 terminal 和浏览器间来回切换。同一模板在 serve 和 file:// 下自动切换（Progressive Enhancement）。
四 Lane 历史
不同 capability level 的 run（git/patch/compare/staged）分 lane 展示，不混在一起比。降级 run 自动标记，不会拉低整体质量评估。
三层锁
CLI 和 serve 可以同时运行，写入安全由三层保护（portalocker 文件锁 → SQLite WAL + busy_timeout → asyncio.Lock），无需用户手动协调。
Graphify 智能缓存
7 态新鲜度判断 + 4 值投影，squash/cherry-pick/rebase 后不会误判为过期，减少无谓的 context 重建开销。
i18n 全链路
浏览器自动检测语言 + Topbar 手动 zh/EN 切换。LLM prompt 用 OUTPUT_LANGUAGE 指令前缀，生成中文或英文 lesson。SRS 卡片保留创建时语言，不重翻译。10 款竞品无一支持学习笔记多语言。
安全 6 类全覆盖
不仅脱敏 diff 正文，还覆盖文件名、commit message、Graphify 标签、LLM 输出、VCR 录制。统一 redaction_pipeline() 入口，raw → scan → redact → 才能 log/cache/model/render。
30 Corner Cases 全闭合
经多轮三模型审查（Claude+Codex+Gemini，Gemini 429 时 Claude 兜底），从 Quiz staleness 到 Archive bomb，从 locale 归一化到 z-index 层级，30 个 corner case 全部有具体工程闭合方案。详见 .claude/team-plan/corner-cases-closure-8.md 和评估报告。

最大威胁：CodeRabbit

12 款竞品里唯一威胁等级"中高"。它已经占住"PR Review + 架构图 + 摘要"，如果加上 SRS/learning，就会直接进入 AhaDiff 的领地。

$24 / 用户 / 月

跟 AhaDiff 的重合点：都处理 diff、都生成说明。
知返如何反击
  • 抢占 Claude Skill 生态入口 — 发布到 awesome-claude-skills（55k stars），让用户装一个 skill 就得到 AhaDiff。
  • 强调 Local-First 个人工具定位 — CodeRabbit 是企业 SaaS，AhaDiff 是个人本地。数据不出本机是卖点，不是借口。
  • Claim 五状态 + SRS 组合 — CodeRabbit 定位为 code review 工具（服务 reviewer），AhaDiff 定位为 diff learning 工具（服务学习者），用户群和优化方向不同。
  • 棘轮机制差异化 — CodeRabbit 的目标是"找出所有问题"；AhaDiff 的目标是"笔记质量单调递增"，完全不同的优化目标。
