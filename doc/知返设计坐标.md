## 知返设计坐标

### 定位

> "Code Wiki 解释仓库，知返解释这次改动；而且每句话都能回到代码证据。"
> -- `doc/知返ahadiff改名后的后续方案.md`:L109-113

知返是一个 **local-first 的 verified diff learning layer**，把 AI 工具写出的 git diff 变成带代码证据链的学习笔记、概念图谱、主动回忆测验、SRS 复习卡和质量棘轮记录。(`CLAUDE.md`:L5-7)

### 七层架构

| # | 层 | 输入 | 输出 | 文档来源 |
|---|---|---|---|---|
| 1 | Diff Capture | git diff / PR patch / staged changes | sanitized raw patch, changed symbols, hunks | `改名方案.md`:L67-72 |
| 2 | Context | local repo files, graphify-out/GRAPH_REPORT.md, prior wiki index, SPEC.md, .ahadiffignore | 过滤后的上下文包 | `改名方案.md`:L74-80 |
| 3 | Lesson Generation | generator_prompt.md, level(beginner/intermediate/senior), mode(full/hint/quiz-only) | lesson.md + claim extraction | `改名方案.md`:L82-88 |
| 4 | Verification | claims.jsonl | deterministic verifier 结果 + LLM judge 结果 + uncertainty report | `改名方案.md`:L90-96 |
| 5 | Ratchet | evaluator.py(immutable), results.tsv | keep / discard / rewrite 决策 + weakest-dimension-first repair | `改名方案.md`:L98-104 |
| 6 | Learning | quiz.md / quiz.jsonl, SRS review | section helpfulness, forgetting-risk dashboard | `改名方案.md`:L106-112 |
| 7 | Wiki + UI | index.md, commit pages, concept graph | evidence inspector, dashboard, shareable cards | `改名方案.md`:L114-121 |

### 三文件契约

- **program.md**: 自然语言状态机，人类写，agent 解释执行。定义 LOOP FOREVER 循环（branch/edit/commit/eval/grep/ratchet/log），含 NEVER STOP 指令。概念改编自 autoresearch 的 program.md（原版控制 train.py 实验循环）。 | 来源: `设计思路.md`:L130-160, `SOURCE-CODE-VERIFICATION-REPORT.md`:L34-44
- **evaluator.py**: IMMUTABLE 评估器，agent 不可修改。函数 `evaluate_wiki(wiki_path) -> float` 返回 wiki_score in [0,1]，stdout 协议固定 `^wiki_score:` 前缀。对应 autoresearch 的 prepare.py（含 `evaluate_bpb`，行 343-365）。 | 来源: `设计思路.md`:L162-180, `VERIFICATION-REPORT.md`:L53-54
- **generator_prompt.md**: agent 唯一可改的文件，用来提升 lesson 质量。对应 autoresearch 的 train.py（改 Python 代码），AhaDiff 改 Markdown prompt。 | 来源: `设计思路.md`:L146, `VERIFICATION-REPORT.md`:L54, `改名方案.md`:L225-235

### Claim Verifier

四种 claim 状态（`改名方案.md`:L487-492）：

| 状态 | 判定方式 |
|---|---|
| **verified** | 能回到 diff hunk，file:line 存在于 diff 中，函数名真实存在，因果合理 |
| **weak** | 有代码线索但因果不完整 |
| **not_proven** | diff 无法证明（如性能提升、作者意图） |
| **contradicted** | 与代码冲突 |

证据绑定方式：每条 claim 绑定 `source_hunks[{file, start, end, hunk_hash}]` + `changed_symbols[]` + `concepts[]`。deterministic verifier 先做（file/line/hunk/symbol 检查 + risky words 扫描），LLM judge 只负责因果合理性和教学价值判断。(`改名方案.md`:L457-485, `kickoff.md`:L194-195)

### 棘轮机制

- **commit（keep）**: 新 wiki_score 严格高于旧值 --> `git commit` 保留，branch advance (`设计思路.md`:L100-108)
- **rollback（discard）**: 新 score >= 旧 score --> `git reset --hard HEAD~1`（默认）或 `git revert HEAD`（`--audit` 模式） (`设计思路.md`:L100-108, `改名方案.md`:L324-330)
- **Phase 2.5（探索性重写）**: 连续 2 个优化目标在首轮即无增益时触发。`git stash` 当前最优版 --> 从头重写 --> 评估 --> 超过则采纳，否则 `git stash pop`。**完全来自 darwin-skill，非 autoresearch**。(`VERIFICATION-REPORT.md`:L13-14, L81-86)

指标：`wiki_score`（higher-better，[0,1]），配套 `results.tsv` 10 列 TAB 分隔（timestamp / run_id / head_sha / prompt_version / rubric_version / overall / verdict / status / weakest_dim / note）。(`VERIFICATION-REPORT.md`:L222, `kickoff.md`:L33)

### 8 维 rubric

> 归因：AhaDiff 的 8 维是完全自研体系。darwin-skill 有 8 维但评估 skill 文件质量；SkillCompass 原版 6 维也评估 skill 文件质量。AhaDiff 评估的是学习笔记质量。(`设计思路.md`:L3, `VERIFICATION-REPORT.md`:L218)

| # | 维度 | 权重 | 硬门禁 |
|---|---|---:|---|
| 1 | accuracy（准确性） | 20 | <15 = FAIL |
| 2 | evidence（证据链完整度） | 18 | -- |
| 3 | diff_coverage（diff 覆盖度） | 14 | -- |
| 4 | learnability（可学性） | 14 | -- |
| 5 | quiz_transfer（测验迁移） | 10 | -- |
| 6 | spec_alignment（与 SPEC 一致） | 10 | -- |
| 7 | conciseness（简洁度） | 8 | -- |
| 8 | safety_privacy（安全隐私） | 6 | -- |
| | **总计** | **100** | |

三档 verdict：PASS >= 80 / CAUTION 60-79 / FAIL < 60 或准确性硬 gate 失败。SkillCompass 原版阈值 70/50，AhaDiff 调高为 80/60。(`kickoff.md`:L33, `VERIFICATION-REPORT.md`:L116-118, `改名方案.md`:L338-342)

### 学习闭环数据流

```
git diff --> Diff Capture(patch/hunks/symbols)
  --> Context Layer(repo files + graphify + .ahadiffignore filter)
    --> Lesson Generation(generator_prompt.md --> lesson.md + claims extraction)
      --> Verification(claims.jsonl: verified/weak/not_proven/contradicted)
        --> Ratchet(evaluator.py --> score.json --> keep/discard/rewrite)
          --> Learning(quiz.jsonl --> SRS review.db --> section helpfulness)
            --> Wiki + UI(index.md + concepts.jsonl 增量积累 + concept graph)
```

关键闭环：section helpfulness Δ（score_with_section - score_without_section）反哺棘轮，Δ <= ε 的章节被压缩或删除（SKILL0 思想，原版 file 级，AhaDiff 扩展到 section 粒度）。(`改名方案.md`:L145-165, `VERIFICATION-REPORT.md`:L196-206)

### Local-first 边界

**留本机**：所有源码、diff、lesson.md、claims.jsonl、quiz.jsonl、score.json、results.tsv、review.db、graph.json、index.md、audit.jsonl。`.ahadiff/` 目录为核心存储。(`改名方案.md`:L3, `最终方案.md`:产品原则段)

**出本机**：仅 LLM API 调用时发送的 prompt（含 diff 片段）。受以下约束控制：
- `.ahadiffignore` 过滤路径（类比 .gitignore）(`设计思路.md`:配置层)
- `safety/redact.py` 扫描并脱敏 secrets (`kickoff.md`:L72-87)
- `config.toml` 中 `offline_only = true` 可强制仅用本地模型 (`设计思路.md`:配置层 privacy 段)
- `audit.jsonl` 记录每次 API 调用去向（文件、token、cost、provider）(`改名方案.md`:Settings 段)

### CLI/UI 形态

**CLI 命令动词** (`改名方案.md`:L495-525)：
```
ahadiff plan "add OAuth login"        # 事前 spec
ahadiff learn HEAD~1..HEAD --open     # 核心学习
ahadiff learn --staged / --pr 123     # 其他 diff 源
ahadiff verify <run_id>               # 证据检查
ahadiff improve <run_id> --rounds 6   # 棘轮优化
ahadiff quiz <run_id>                 # 主动回忆
ahadiff review / review --weak        # SRS 复习
ahadiff graph / graph import          # 图谱
ahadiff install claude/codex/cursor   # agent 安装
ahadiff card / export                 # 分享
```

**UI 视图** (11 页面 x 3 风格 = 33 界面)：Landing / Runs Dashboard / Lesson Reader / Diff+Evidence Viewer / Ratchet Lab / Socratic Quiz / SRS Review / Settings / Onboarding / Agent Skill Hub / Learning Graph Explorer (`frontend_design_v1.1.md`:L70-83)

**技术栈**：Python 3.11+, typer, rich, pydantic, jinja2, httpx, pyyaml; 首版 viewer 用 Jinja2 静态 HTML; SQLite 存 SRS review (`kickoff.md`:L40-43)
