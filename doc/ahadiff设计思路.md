| 维度 | 权重 | 评分标准 | |---|---|---|---| | 1 | Frontmatter 质量 | 8 | name 规范、description 含做什么+何时用+触发词、≤1024 字符 | | 2 | 工作流清晰度 | 15 | 步骤明确、有序号、每步有输入/输出 | | 3 | 边界条件覆盖 | 10 | 异常处理、fallback 路径、错误恢复 | | 4 | 检查点设计 | 7 | 关键决策前用户确认、防自主失控 | | 5 | 指令具体性 | 15 | 不模糊、有参数/格式/示例、可直接执行 | | 6 | 资源整合度 | 5 | references/scripts/assets 路径可达 | | 7 | 整体架构 | 15 | 层次清晰、不冗余不遗漏 | | 8 | 实测表现 | 25 | 用 test prompt 跑一遍,with-skill vs baseline 对比 | **keep/revert 机制**:**`git revert HEAD` 而非 `git reset  --hard`**(保留完整历史!),初始化 `git checkout -b auto-optimize/YYYYMMDD-HHMM`。这跟  autoresearch 的 reset 哲学不同——darwin 偏审计追溯,autoresearch 偏 branch  整洁。**AntiVibe 建议 hybrid**:默认 `git reset`(干净),加 `--audit` 标志切换到 `git  revert`(审计)。 **before/after 表格**:ASCII box-drawing 字符(`┌ ─ ┬ ┐ │`),LLM  直出文本,无代码渲染库。AntiVibe 可用 `rich.table.Table` 产出更漂亮版本,但保留紧凑风格。 **evolve 循环关键补充**:Phase 2.5 探索性重写（**完全来自 darwin-skill，非 autoresearch**）——连续 2 个 skill 在 round 1 就 break 时,`git  stash` 当前最优版,从头重写,超过 stash 则采用,否则 `git stash pop`。**这是防止 hill-climbing  卡死的关键兜底**,AntiVibe 必须抄。 **子 agent 独立评分避免自评偏差**:darwin 明确要求"一个带 skill 跑,一个不带 skill 跑",**由独立子  agent 重新打分**。AntiVibe 对应:**生成用 Sonnet,评估用 Haiku 或 GPT-4o-mini**,强制跨模型评估。 ### 2.4 SkillCompass(Evol-ai)真机借鉴 **实测状态**:https://github.com/Evol-ai/SkillCompass,约 40 star(自报),launch  2026-03,针对 Claude Code skill 做 6 维诊断。需 Node.js v18+ 和 Claude Opus 4.6。 **6 维 + 三档门限**: - **D1 Metadata/Description**(frontmatter 清晰度) - **D2 Trigger**(何时激活;`not_for` 拒绝边界——80% 生态 skill 缺失) - **D3 Security**(**硬 gate**:单维 Critical 强制 FAIL,无覆盖) - **D4 Functional**(LLM 指令质量:错误恢复、边界、输出规格) - **D5 Supersession**(skill 对比 base model;model 更新后是否仍胜出) - **D6 Multilingual/Usage 信号** **权重**:60% 静态分析 + 40% 反馈信号。**CI exit  code**:0=PASS,1=CAUTION,2=FAIL。**Safety rollback**:任一维度改进后下降 >2 分 →  丢弃改进。**Paired-dimension improvement**:D1+D2 若同时 ≤5 一起改(共享 frontmatter  层)。 #### AhaDiff 8 维自研 Rubric（非来自 SkillCompass 或 darwin-skill）
