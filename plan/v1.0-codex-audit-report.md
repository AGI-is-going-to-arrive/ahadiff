## D2. 方案合理性

| 方案 | 评估 | 替代方案 | 建议 |
|---|---|---|---|
| concepts JSONL → SQLite | 方向正确；10k JSONL 仍可用，但 100k、FTS、weak concepts、Graphify linking 需要 SQLite | SQLite 作为 truth source，JSONL 只做导出/兼容缓存 | 采纳，但补 migration rollback、dual-write、branch ancestry 预计算 |

| tree-sitter optional | 作为增强合理，但不是当前实现；0.5 天估算偏低 | optional extra + lazy import + doctor capability report + fallback tests | 修改后采纳 |
| CLI 冷启动懒加载 | 合理；research-v2 指出 `llm` import 约 220ms，但 `core.config` 也占大头 | 命令级 import，`--help/--version/config/doctor` 不 import LLM/provider | 采纳，并加启动性能回归 |
| 前端路由拆分 | 当前 bundle gzip 91.38KB，不是立即瓶颈；加入 Search/Graphify/Zod 后有价值 | 只 lazy-load 重页面/重依赖，shell eager | 修改后采纳 |

## 最终判定

### 是否可以基于这些 research 文档制定可执行的 v1.0 plan？

- [ ] GO: 文档准确、完整、可直接用于 plan
- [x] CONDITIONAL GO: 需修正 9 项后可用于 plan
- [ ] NO GO: 存在重大缺陷需返工

### 必须修正的条件项

1. 把 API route 口径改成“21 个显式业务 API route + 1 个 `/api` catchall + 1 个 `/healthz`”，或明确“22 concrete API endpoints”的统计边界。
2. 把 tree-sitter 标为 v1.0 待实现 optional enhancement，不得写成当前实现。
3. 把 Graphify 深度集成标为 v1.0 待实现；当前仅有基础 detect/import/status/refresh。
4. 修正后端/前端工期口径：区分 person-days 与 parallel calendar days。
5. 修正 Competitors deferred Include 计数：当前是 6 项，不是 5 项。
6. 更新 V6 gap 文档：Diff、ConceptGraph、tokens、Review scaffolding 等已有漂移。
7. 把 Windows reparse 防御缺口加入 v1.0 hardening：serve finalized artifacts、eval digest、review/usage SQLite leaf。
8. 把 `/api/auth/token` 无认证返回 token 的风险纳入 plan；至少明确 threat model 和浏览器/CORS/本地进程边界。
9. 给 performance baseline 非 bundle 项补复现脚本或降级为 ad-hoc baseline。

**同步状态**：这 9 项已映射到 `plan/v1.0-research-v2.md`、`.claude/team-plan/v10-research-v2.md`、`.claude/team-plan/v10-scope-collection.md`、serve/performance/backend/frontend/V6 gap research 文档。后续 implementation plan 应直接采用更新后的 v1.0 Full Scope 口径。

### 遗漏的约束或风险

- `usage.sqlite` 作为 global derived governance 仍可能跟随 symlink；虽然不是 truth source，但会写本机状态目录。
- `review.sqlite` 只拒绝 POSIX symlink，Windows reparse point 仍需统一 helper。
- `serve` middleware 允许 `::1` host，但 CLI 不允许绑定 `::1`；IPv6-only loopback 是可用性风险。
- LLM provider `base_url` SSRF 不是 patch URL 下载器级别的防护；如果未来允许不可信输入控制 provider URL，必须补 DNS/IP 检查。
- Phase 2.5 事件语义需要写清楚：当前是 `targeted_verify + phase25 note`，不是 `phase25_rewrite` status。

### 工期调整建议

- 后端用 **42-53 person-days** 做 v1.0 Full Scope 估算。
- 前端按 **18-23 person-days** 管理 full V6 parity + Graphify UI + tests。
- v1.0 **不拆 Core/Full**。B0-B6 / F1-F5 只是同一 v1.0 milestone 内的依赖顺序和并行调度，不代表降级交付范围。
