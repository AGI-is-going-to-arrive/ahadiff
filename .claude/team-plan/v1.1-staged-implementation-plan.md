#### Gate 0.1: Git revision option-boundary hardening

**状态**：`DONE`

**范围**：
- `src/ahadiff/git/repo.py`：`_resolve_commitish`、`run_git`、`resolve_ref_range`
- `src/ahadiff/git/capture.py`：`_run_git_patch_text`、所有 `git show`/`git diff` 调用

**具体改动**：
1. 在所有接受 untrusted revision 的 git 调用中，在 revision 参数前插入 `--end-of-options`
2. untrusted pathspec 继续使用 `--` 分隔，并保持 literal pathspec / repo-relative 校验；不要用 `--end-of-options` 代替 pathspec separator
3. `_resolve_commitish` 入口添加 leading-dash 拒绝：`if revision.startswith("-"): raise InputError(...)`
4. `resolve_ref_range` 对 `..` 分割后的每段做同样检查

**验证命令**：
```bash

```

**通过标准**：
- 新增 ≥3 个测试用例：`--upload-pack=...`、`-rf`、`--exec=...` 均 raise `InputError`
- 合法 revision（`HEAD`, `main`, `abc123`, `v1.0`, `feature/x`）不受影响
- 全量 `tests/unit/test_git_capture.py` 通过
