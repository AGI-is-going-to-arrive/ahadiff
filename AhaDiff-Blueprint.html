关键语义：组合模式与窗口 diff

--staged --unstaged / --since / --include-untracked
--staged --unstaged 组合
不拼接两份 patch。直接执行 git diff HEAD --no-ext-diff，产出单一 patch 以 HEAD 为基准，包含 index 和 worktree 的全部改动。

source_kind = git_unstaged
metadata: combined_mode=true
--since "2h ago" 语义
连续 ancestry 窗口 diff（始终产出单一自洽 patch）。

1. git rev-list --first-parent --since 获取命中 commit 列表
2. 取最早命中 commit 的父节点为 base
3. 执行 git diff base..HEAD

中间有他人 commit 仍包含在窗口内（避免语义断裂）。
--include-untracked
配合 --unstaged 或 --staged --unstaged 使用。

通过 git ls-files --others --exclude-standard 列出新文件，当作全新增 diff 纳入。

按 .ahadiffignore 过滤 + file_count_exceeded 截断。默认不含 untracked（CLI warn 提示）。

Q1: Diff 是怎么检测的？和 IDE 的 LSP/AST 有什么不同？

Codex 验证
AhaDiff 的方式 · Git Unified Diff
@@ -68,7 +68,29 @@ class ApiClient - async request(path) { - return fetch(path) + async request(path, opts) { + for (let i = 0; i<=max; i++) + try { return await fetch() }
看到的是：哪几行被加了（+）、删了（-）、在哪个文件、有限的上下文（hunk header 会提示函数名）
不擅长的：稳定、结构化地理解 symbol 关系、调用链、语义影响
语言无关 零依赖 半成品代码也行 隐私友好
IDE 的方式 · LSP + AST
ClassDeclaration: ApiClient MethodDefinition: request ├─ params: [path, opts] └─ body: ForStatement └─ TryStatement └─ AwaitExpression: fetch()
看到的是：代码的结构（函数、类、参数、调用关系）
看不到的是：这次 commit 到底改了什么（它不做 diff）
需按语言安装 需语言服务 重量级
AhaDiff 的策略：用 git diff 作为真相源（"到底改了什么"），由 LLM 补全语义理解（"为什么这样改"）。未来可考虑引入 Tree-sitter AST 作为增强层（提升 symbol 提取精度），但永远不替代 diff。（AST 增强尚未纳入正式方案）

一句话：IDE 用 AST 理解代码结构，AhaDiff 用 diff 锁定变更事实 + LLM 理解变更含义。两者互补，不冲突。

Q7: 复习卡片怎么调度？为什么用 FSRS-6 而不是 SM-2？

FSRS-6 · DSR 三分量模型
AhaDiff 采用 FSRS-6（Free Spaced Repetition Scheduler）作为 SRS 调度算法，使用 py-fsrs 库（v6.3.1+）。FSRS-6 基于 DSR 三分量记忆模型（Difficulty / Stability / Retrievability），Anki 23.10 起可手动启用，RemNote 已内置。
对比维度SM-2（1987，已放弃）FSRS-6（2024，AhaDiff 采用）
算法基础1987 年固定 EF 衰减公式2023-2024 DSR 三分量记忆模型
核心参数2 个（ease_factor, interval）21 个（opaque weights JSON array，不写死数量）
遗忘曲线指数衰减（不精确）幂律衰减（empirically validated）
自适应无Optimizer 自动训练个人参数
复习效率基线减少 20-30% 复习量（RemNote 经验值，中置信）
Python 库需自行实现pip install fsrs（py-fsrs v6.3.1）

Q8: 所以 AhaDiff 到底是什么？一句话版本

最终定位
AhaDiff 是一个只读的本地 CLI。

它读取 AI 写出的 git diff（或 patch 文件），
把每句解释都绑定到代码证据，
标出 AI 说错的地方，
生成 Lesson / Quiz / 复习卡，
然后自动评分，越来越好。

它不改你的代码。
它教你理解你的代码。
