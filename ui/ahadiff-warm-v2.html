After · Aha Lesson（excerpt）

这次改动把 request() 从单次 fetch 改成了 指数退避 + 抖动的重试循环。重试次数由 opts.retries 配置，默认 4 次；每次失败后延迟 200·2^i 毫秒加上一个均匀分布的抖动。

为什么重要？同步重试容易引发 Thundering Herd；抖动能把大量客户端的重试点错开。

Not proven by this diff: 真正的性能提升不能仅由代码得出。

TL;DR

这次改动给 ApiClient.request 加上了 指数退避 + 抖动 的重试循环，默认重试 4 次；失败会抛出原始错误。它解决的是短暂网络抖动导致请求失败的问题，但不解决幂等性——GET 以外的请求如果你不显式标注 idempotencyKey，重试是危险的。

What changed

新增了 ReqOpts 可选参数；默认行为从“一次请求”变成“最多 5 次尝试，逐次退避 200·2^i 毫秒 + 随机抖动”。没有默认 idempotencyKey；没有对 HTTP 状态码做分类（5xx vs 4xx）；没有引入 AbortController。

Why it matters

在面向多租户 API 时，单次 fetch 遇到瞬断会把失败原样抛给用户。加入退避可以降低短暂网络抖动造成的可见失败；具体改善幅度需要线上指标或 benchmark 证明（见 Not Proven）。而抖动是在大量客户端同时重试时避免 Thundering Herd 的关键——这也是这次 diff 最值得记住的一条。

Walkthrough by hunk

hunk a7f2 · L76–L83：这是重试循环本体。注意 i <= max 意味着总尝试次数是 max + 1，这是一个经常被记错的细节。backoff + jitter 而不是 backoff * (1 + jitter)，两者在长尾分布上并不相同。

Concepts you just used

Exponential backoff · Jitter · Idempotency (未触达) · Retry budget · Thundering Herd. 其中 jitter 是本 lesson 首次引入，会进入 SRS 复习队列。

Misconceptions

很多人以为加重试一定更可靠——对非幂等请求反而会放大副作用。常见的另一错觉：抖动没有必要。从数学上看，没有抖动时大量同步客户端在第 n 次重试会在同一毫秒聚合。

Not proven by this diff

  • 性能提升幅度（“吃掉 X% 瞬断” / “延迟降低 Y%”）——无 benchmark。
  • 作者真正的意图——无法从代码独立验证。
  • 安全影响——这次不涉及 token / 敏感路径，因此也无法说“安全”。
  • POST 请求重试的安全性——已 contradicted（见 c020）。

QUESTION · III of V
在当前实现里，for (let i = 0; i <= max; i++) 的循环条件意味着最多会发起多少次 fetch？
A
max 次
等于 opts.retries 默认 4 次
B
max + 1 次
因为 i <= max 包含 0 和 max
C
2^max 次
D
无限次，直到成功
Explanation · 正确
i 从 0 循环到 max 共 max + 1 次；只有第 max 次捕获异常时才 throw。这意味着默认情况下最多 5 次 fetch，不是 4 次——这也是经常被记错的细节。

- user asks to "学一下这个 commit" / "explain this PR"

1. lesson.md (prose, sections defined in references/output-contract.md)
2. claims.jsonl (one claim per line, status ∈ verified|weak|not_proven|contradicted)

- Accuracy < 14/20
- Evidence < 10/15
- Contradicted claims > 0
- Any prompt injection unresolved

# AGENTS.md · knowledge-return

When a commit lands, the `ahadiff` skill SHOULD:
1. Read the diff via `git diff HEAD~1..HEAD`.
2. Ground every claim to file:line.
3. Prefer short claims over long prose.
4. Never upload .env, *.pem, or any secret.
5. If a claim cannot be grounded, mark status: not_proven.

## Triggers (always-on)
- on commit to main → ahadiff learn
- on PR opened     → ahadiff learn --against SPEC.md
- on SPEC.md edit  → ahadiff plan --refresh
