Lots of people now use AI assistants to write code. Cursor, Claude Code, Copilot. You can change dozens of files in a day. You committed, the code runs, but do you really know what changed? This is the knowledge gap of vibe coding. AI implements the feature, but the changes never become your own knowledge. Next time you face a similar problem, you still have to ask AI again. AhaDiff exists to solve this.

Using AhaDiff is simple. AhaDiff is not published on PyPI yet. In a source checkout, install the CLI with uv tool install --editable dot, or run directly with uv run ahadiff. After installation, go to your project directory and run ahadiff init to initialize. Then, after AI changes your code, run ahadiff learn with the commit range you want to study. For example, HEAD tilde one to HEAD means the latest commit. Add --staged to learn staged changes, or --unstaged to learn working directory changes. You can also use --changed-path to learn a specific file. After learning, run ahadiff serve to open the web UI and view results. The Guide page lists all commands including quiz, review, verify, improve, plus advanced database maintenance and concept management commands.

AhaDiff does not just generate a commit summary. Its core idea is learning with evidence: every sentence in the lesson must point to a specific file and line number. Open the Diff page and you see the code changes with colored dots marking lines mentioned in the lesson. Click a dot and the inspector panel shows the explanation and code snippet. Each conclusion has five verification statuses: verified, weak, not proven, contradicted, and rejected. What you learn is not hallucination but real knowledge traceable to code.

The Lesson page shows the full lesson for this change. AhaDiff generates three tiers: full for deep learning, hint for key points only, and compact for a minimal summary. The top of the lesson shows the score and pass status. You can click View Score Details to see the scoring breakdown, or View LLM Review to see the model's semantic review. These scores help you judge whether the generated lesson is reliable.

Just reading lessons is easy to forget. AhaDiff helps you remember in two ways. The Quiz page shows multiple-choice questions where you must think of the answer yourself. This is called active recall, much more effective than passive reading. After the quiz, knowledge points become review cards. The Review page uses a spaced repetition algorithm to schedule reviews right before you forget. Each card is rated by mastery level, and the system adjusts the next review interval automatically. AhaDiff scores each learning session on 8 dimensions, out of 100. Accuracy checks whether lesson content is correct. Evidence checks if every claim has a code source. Diff coverage checks if the lesson covers the main changed files. Learnability checks if the lesson is suitable for learning. Quiz transfer checks if quiz questions match the lesson. Spec alignment checks if your specified learning requirements are met. Conciseness checks if the length is reasonable. Safety and privacy checks for sensitive information leaks. Accuracy and evidence have hard gates. Below the threshold means automatic failure. You can view each dimension's score and the LLM review on the Run Detail page's Score and Judge tabs.

AhaDiff does not discard what you learned. Each session extracts new concepts accumulated in the concept ledger. The ledger page shows all knowledge points, their source files, and health status. Switch to the graph view to see the relationship network. Nodes are knowledge points, edges are their connections. This graph grows as you learn more code changes. The data here is from my local testing. You need to run learn in your own project to gradually build your own concept graph. By the way, the concept accumulation design draws from Karpathy's LLM Wiki idea and the Graphify code graph tool, landed as a diff-level learning overlay.

The Dashboard shows all your learning sessions, score trends, and cards due for review. You can see which sessions passed or failed, and whether scores are trending up or down. The Settings page lets you configure your own LLM provider. It supports OpenAI, Gemini, Anthropic and other formats. Just fill in the endpoint and your key. You can also run ahadiff doctor to check your environment, or ahadiff config show --resolved to see the current configuration. AhaDiff supports project-level integration with 13 AI coding tools including Claude Code, Cursor, Copilot, and Codex. One ahadiff install command writes the guidance file for each tool. All learning records are saved in your project directory, and model calls are configured and managed by you.

To summarize. AhaDiff solves the core problem: AI changed your code, but you did not learn from it. It uses evidence-based learning to ensure what you learn is real knowledge traceable to code. In a source checkout, install with uv tool install, run ahadiff init to initialize, configure your model provider, then run learn on a recent change. Open the web UI to read lessons, check evidence, take quizzes, and review periodically. AhaDiff also supports project-level integration with 13 AI coding tools, so your AI assistants know how to use AhaDiff too. Let AI write code. Let AhaDiff return the knowledge to you.
